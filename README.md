## Общее описание

**TashGresOpto** - система оперативного учета и расчета технико-экономических показателей работы ТЭС (Ташкентская ГРЭС).

### Назначение системы

Система предназначена для:
- Ежедневного расчета экономии/перерасхода топлива по вахтам
- Учета выработки электроэнергии
- Расчета пережогов топлива
- Формирования сводных отчетов
- Контроля технологических параметров работы блоков

### История

Оригинальная система была разработана в среде KARAT/M (DOS-based СУБД) в 1990-2000-х годах. Данная версия является полной миграцией на современные технологии (C# .NET 8.0, Windows Forms, SQLite) с сохранением всех формул и функционала.

## Архитектура системы

### Технологический стек

- **Платформа:** .NET 8.0
- **UI:** Windows Forms
- **База данных:** SQLite + Entity Framework Core
- **Архитектура:** Многослойная (Layered Architecture)

### Структура проекта

```

├── TashGresOpto.UI/          # Пользовательский интерфейс
│   ├── Forms/                 # Формы и пользовательские контролы

│   └── Program.cs             # Точка входа
├── TashGresOpto.Core/        # Бизнес-логика
│   ├── Calculators/           # Калькуляторы для расчетов
│   ├── Services/              # Сервисы (агрегация, отчеты)
│   └── Models/                # Модели данных
├── TashGresOpto.Data/        # Слой данных
│   ├── Entities/              # Сущности БД
```

### Принципы проектирования

1. **Разделение ответственности (SoC)** - каждый слой отвечает за свою задачу
2. **Dependency Injection** - внедрение зависимостей через DI контейнер

## Модули системы

### 1. BAXTA - Ежедневный расчет по вахтам

**Назначение:** Расчет экономии/перерасхода топлива по каждому энергоблоку и вахте (смене).

#### Функциональность

- Ввод технологических параметров для каждого блока и вахты
- Расчет 7 составляющих экономии/перерасхода топлива (ЭКСУТ1-7)
- Вывод результатов по блокам и сменам
- Просмотр графика работы вахт
- Сохранение данных в БД

#### Используемые таблицы

- **BaxtaParams** - технологические параметры
- **BaxtaCounter** - данные счетчиков и приборов
- **Baxta** - график работы вахт

#### Технологические параметры

| Параметр | Обозначение | Единицы | Диапазон | Описание |
|----------|-------------|---------|----------|----------|
| THW | Температура холодной воды | °C | 0-50 | Температура сырой воды на входе в котел |
| TUG | Температура уходящих газов | °C | 50-300 | Температура газов на выходе из котла |
| O2 | Содержание кислорода | % | 0-21 | Объемное содержание O₂ в уходящих газах |
| TK | Температура конденсата | °C | 0-100 | Температура конденсата турбины |
| POP | Давление острого пара | кгс/см² | 50-200 | Давление свежего пара перед турбиной |
| TOP | Температура острого пара | °C | 400-600 | Температура свежего пара |
| TPP | Температура промперегрева | °C | 400-600 | Температура пара после промперегрева |
| TPW | Температура питательной воды | °C | 100-300 | Температура воды перед котлом |
| DRO | Дросселирование | % | 0-20 | Степень дросселирования пара |
| PWD | Признак ПВД | 0/1 | - | 0-откл., 1-вкл. (подогреватель высокого давления) |
| TIM | Время работы | час | 0-24 | Время работы блока за смену |

#### Коэффициенты

- **MPRIS** - коэффициент присосов воздуха
- **MTCB** - коэффициент тепловой схемы блока
- **MKF1, MKF2** - коэффициенты собственных нужд
- **MKL1, MKL2** - коэффициенты питательной воды

### 2. PEREJEG - Пережоги топлива

**Назначение:** Учет пережогов (перерасхода) топлива по блокам.

#### Функциональность

- Ввод данных о пережогах по 12 блокам
- Расчет 16 показателей пережога
- Печать суточных и накопленных отчетов
- Просмотр расчетных дат

#### Показатели пережога (16 типов)

1. **ПУГ** - Потери с уходящими газами
2. **ВАК** - Потери от химнедожога
3. **ДОП** - Дополнительные потери
4. **ТОП** - Потери от температуры острого пара
5. **ТПП** - Потери от температуры промперегрева
6. **СН** - Потери на собственные нужды
7. **ТПВ** - Потери от температуры питательной воды
8. **ПУР** - Потери от уровня работы
9. **ВСЕГО** - Суммарные потери
10. **ВЫР** - Выработка электроэнергии
11. **НАГР** - Средняя нагрузка
12. **УР** - Удельный расход топлива
13-16. **Резервные показатели**

#### Используемые таблицы

- **Perejeg** - данные пережогов
- **PerejegType** - справочник показателей

## Формулы и расчеты

### Модуль BAXTA - Расчет экономии/перерасхода топлива

#### ЭКСУТ1 - Потери с уходящими газами (присосы воздуха и O₂)

```
ЭКСУТ1 = 3.5 × (RRF - RRN) × WY

где:
RRN = 5.15 + 0.0083 × N + 0.022 × (15 - THW)

RRF = MPRIS × 0.01 × √(160 / N)
RRF = RRF + (21 - 0.1 × O2) / (21 - O2)
RRF = (RRF × 3.5 + 0.6) × (TUG - THW) × 0.01

N   - нагрузка (МВт)
WY  - выработка (МВт·ч)
THW - температура холодной воды (°C)
TUG - температура уходящих газов (°C)
O2  - содержание кислорода (%)
MPRIS - коэффициент присосов воздуха
```

**Физический смысл:** Чем выше температура уходящих газов и содержание O₂, тем больше потери с дымовыми газами. Формула учитывает:
- Присосы воздуха (MPRIS)
- Избыток кислорода (O2)
- Перегрев уходящих газов (TUG - THW)
- Зависимость от нагрузки блока

#### ЭКСУТ2 - Потери с химическим недожогом

```
ЭКСУТ2 = 0.68 × WY × (TK - RR)

где:
RR = MTCB + 0.07 × N + (20 + N) / (MTCB + 17)

TK    - температура конденсата (°C)
MTCB  - коэффициент тепловой схемы блока
N     - нагрузка (МВт)
```

**Физический смысл:** Потери от неполного сгорания топлива. Зависят от температуры конденсата и тепловой схемы блока.

#### ЭКСУТ3 - Дополнительная экономия от нагрузки

```
Если POP < 126 кгс/см², то:

При 80 ≤ N ≤ 117:
  ЭКСУТ3 = 0.148 × (N - 80) × WY

При 117 < N ≤ 160:
  ЭКСУТ3 = 0.127 × (160 - N) × WY

Иначе:
  ЭКСУТ3 = 0

POP - давление острого пара (кгс/см²)
N   - нагрузка (МВт)
```

**Физический смысл:** При пониженном давлении острого пара (POP < 126) возникает дополнительная экономия/перерасход в зависимости от режима нагрузки. Оптимальная нагрузка - около 117-120 МВт.

#### ЭКСУТ4 - Потери от температуры острого пара

```
ЭКСУТ4 = -0.07 × WY × (TOP - 540)

TOP - температура острого пара (°C)
```

**Физический смысл:** Отклонение температуры острого пара от номинальной (540°C) влияет на КПД цикла. Повышение температуры дает экономию, понижение - перерасход.

#### ЭКСУТ5 - Потери от температуры промперегрева

```
RR = 540°C  (номинал)

Если 80 < N < 115:
  RR = 530 + 0.429 × (N - 80)

Если N ≥ 115:
  RR = 540

Если TPP < 540:
  ЭКСУТ5 = -0.056 × WY × (TPP - RR)
Иначе:
  ЭКСУТ5 = 0

TPP - температура промперегрева (°C)
```

**Физический смысл:** Снижение температуры промперегрева ниже нормы приводит к потерям. Норма зависит от нагрузки блока.

#### ЭКСУТ6 - Потери на собственные нужды

```
ЭКСУТ6 = (ESN - RR) × 350

где:
RR = MKF1 × TIM + MKF2 × WY

ESN   - фактические собственные нужды (МВт·ч)
TIM   - время работы (час)
MKF1  - коэффициент СН по времени
MKF2  - коэффициент СН по выработке
```

**Физический смысль:** Отклонение фактических собственных нужд от расчетных (норматива). Нормативные СН рассчитываются как функция времени работы и выработки.

#### ЭКСУТ7 - Потери от температуры питательной воды

```
Если PWD = 0 (ПВД отключен):
  ЭКСУТ7 = (2.98 + 0.0276 × (N - 80)) × WY

Если PWD = 1 (ПВД включен):
  
  При N < 150:
    RR = 170 + MKL1 × N
  
  При 150 ≤ N < 160:
    RR = MKL2 + 0.3 × (N - 150)
  
  При N ≥ 160:
    RR = MKL2 + 3 + 0.2 × (N - 150)
  
  ЭКСУТ7 = -0.066 × (TPW - RR + 0.05 × DRO) × WY

PWD - признак работы ПВД (0/1)
TPW - температура питательной воды (°C)
DRO - дросселирование (%)
MKL1, MKL2 - коэффициенты
```

**Физический смысл:** 
- Если ПВД отключен - постоянный перерасход
- Если ПВД включен - отклонение температуры питательной воды от нормы влияет на экономичность
- Дросселирование также учитывается как дополнительные потери

#### Итоговая экономия и удельный расход

```
ИТОГО = ЭКСУТ1 + ЭКСУТ2 + ЭКСУТ3 + ЭКСУТ4 + ЭКСУТ5 + ЭКСУТ6 + ЭКСУТ7

Удельный расход = ИТОГО / WY  (г/кВт·ч)
```

### Модуль WYRABOTKA - Расчет выработки электроэнергии

#### Расчет выработки по счетчику

```
Разница = ReadingEnd - ReadingStart

Если Разница < 0:
  Разница = Разница + 100000  (переполнение счетчика)

Выработка = Coefficient × Разница × 0.001  (МВт·ч)

Coefficient   - коэффициент трансформации
ReadingStart  - начальное показание (кВт·ч)
ReadingEnd    - конечное показание (кВт·ч)
```

**Физический смысл:** 
- Счетчики учитывают энергию в кВт·ч
- Коэффициент трансформации учитывает соотношение измерительных и силовых цепей
- Переполнение счетчика (5 разрядов) учитывается автоматически

#### Расчет мощности

```
Мощность = Выработка / Часы  (МВт)

Часы - время работы блока (обычно 8 часов за смену)
```

#### Расчет собственных нужд (СН)

```
СН_Разница = SNReadingEnd - SNReadingStart

Если СН_Разница < 0:
  СН_Разница = СН_Разница + 10000  (переполнение счетчика СН)

СН = SNCoefficient × СН_Разница × 0.001  (МВт·ч)

СН_% = (СН / Выработка) × 100
```

**Физический смысл:** Собственные нужды - это энергия, потребляемая вспомогательным оборудованием станции (насосы, вентиляторы, освещение и т.д.). Обычно составляет 5-8% от выработки.

#### Учет замены счетчиков

При замене счетчика в течение суток:

```
Выработка_старый = Coeff_старый × (Показание_замена - Показание_начало) × 0.001
Выработка_новый = Coeff_новый × (Показание_конец - Показание_замена_нового) × 0.001

Выработка_итого = Выработка_старый + Выработка_новый
```

### Модуль O2GOL - Сводный расчет

#### Агрегация данных по блокам

```
Для каждого блока (i = 1..12):
  Расход_топлива[i] = Σ (ЭКСУТ_всех_смен_блока_i)
  Выработка[i] = Σ (Выработка_всех_смен_блока_i)
  Средняя_нагрузка[i] = Σ (Нагрузка_смен) / Количество_смен
  
  Удельный_расход[i] = (Расход_топлива[i] × 1000) / Выработка[i]  (г/кВт·ч)
```

#### Агрегация данных по вахтам

```
Для каждой вахты (j = 1..4):
  Расход_топлива[j] = Σ (ЭКСУТ_всех_блоков_вахты_j)
  Выработка[j] = Σ (Выработка_всех_блоков_вахты_j)
  Средняя_нагрузка[j] = Σ (Нагрузка_блоков) / Количество_блоков
  
  Удельный_расход[j] = (Расход_топлива[j] × 1000) / Выработка[j]
```

#### Итоговые показатели

```
Расход_станции = Σ (Расход_всех_блоков)
Выработка_станции = Σ (Выработка_всех_блоков)
Средняя_нагрузка_станции = Σ (Нагрузка_блоков) / Количество_работавших_блоков
Удельный_расход_станции = (Расход_станции × 1000) / Выработка_станции
```

## Параметры и валидация

### Технологические параметры - допустимые диапазоны

| Параметр | Мин. | Макс. | Типичное | Критичность |
|----------|------|-------|----------|-------------|
| THW (°C) | 0 | 50 | 15-25 | Средняя |
| TUG (°C) | 50 | 300 | 120-180 | Высокая |
| O2 (%) | 0 | 21 | 3-7 | Высокая |
| TK (°C) | 0 | 100 | 30-50 | Средняя |
| POP (кгс/см²) | 50 | 200 | 90-130 | Высокая |
| TOP (°C) | 400 | 600 | 535-545 | Высокая |
| TPP (°C) | 400 | 600 | 535-545 | Высокая |
| TPW (°C) | 100 | 300 | 200-260 | Средняя |
| DRO (%) | 0 | 20 | 0-5 | Низкая |
| PWD | 0 | 1 | 1 | Средняя |
| TIM (ч) | 0 | 24 | 8 | Средняя |

### Валидация данных счетчиков

```csharp
// Номер блока: 1-12
if (blockNumber < 1 || blockNumber > 12)
    return Error("Недопустимый номер блока");

// Номер смены: 1-4
if (shift < 1 || shift > 4)
    return Error("Недопустимый номер смены");

// Выработка: неотрицательная, < 1500 МВт·ч
if (generation < 0)
    return Error("Отрицательная выработка");
if (generation > 1500)
    return Error("Выработка слишком велика");

// СН: 0 ≤ СН ≤ Выработка
if (sn < 0 || sn > generation)
    return Error("Недопустимое значение СН");
```

### Валидация дат

```csharp
// Дата не в будущем
if (date > DateTime.Now.AddDays(1))
    return Error("Дата в будущем");

// Дата не ранее 1990 года
if (date < new DateTime(1990, 1, 1))
    return Error("Дата слишком старая");

// Период не более 365 дней
if ((endDate - startDate).TotalDays > 365)
    return Error("Слишком большой период");
```

### Работа с модулем O2GOL

#### Шаг 1: Запуск модуля

- Для месячного отчета: 1-е число - последнее число месяца
- Максимальный период: 365 дней

#### Шаг 3: Просмотр отчета

Система формирует два отчета:

**Отчет 1: По блокам (12 блоков)**

Таблица 12×10:
- Строки: блоки 1-12
- Колонки: ПУГ, ВАК, ДОП, ТОП, ТПП, СН, ТПВ, ВСЕГО, ВЫР, УР
- Внизу: итого по станции

**Отчет 2: По вахтам (4 смены)**

Таблица 4×10:
- Строки: вахты 1-4
- Колонки: ПУГ, ВАК, ДОП, ТОП, ТПП, СН, ТПВ, ВСЕГО, ВЫР, УР
- Внизу: итого по станции

#### Шаг 4: Анализ результатов

**Интерпретация показателей:**

1. **ПУГ (Потери с уходящими газами)**
   - Положительное значение = перерасход
   - Отрицательное значение = экономия
   - Причины перерасхода: высокая температура газов, избыток O₂

2. **ВАК (Химический недожог)**
   - Обычно небольшое положительное значение
   - Зависит от качества топлива и режима горения

3. **ДОП (Дополнительные потери)**
   - Связаны с режимом работы при пониженном давлении

4. **ТОП, ТПП (Температуры пара)**
   - Отрицательное = экономия (температура выше нормы)
   - Положительное = перерасход (температура ниже нормы)

5. **СН (Собственные нужды)**
   - Положительное = перерасход СН
   - Сравнить с нормой 5-8%

6. **ТПВ (Питательная вода)**
   - Зависит от работы ПВД

7. **ВСЕГО**
   - Суммарная экономия/перерасход
   - Отрицательное = экономия топлива
   - Положительное = перерасход топлива

8. **УР (Удельный расход)**
   - г/кВт·ч
   - Чем ниже, тем экономичнее
   - Норматив: около 340-360 г/кВт·ч

### Сообщения об ошибках

#### Ошибки валидации

**"Температура холодной воды должна быть в диапазоне 0-50°C"**
- Причина: введено недопустимое значение
- Решение: проверьте показания прибора, введите корректное значение

**"Дата не может быть в будущем"**
- Причина: выбрана дата позже текущей
- Решение: выберите сегодняшнюю или прошедшую дату

**"СН не может быть больше выработки"**
- Причина: ошибка ввода СН
- Решение: проверьте показания счетчика СН

**"Начальная дата не может быть позже конечной даты"**
- Причина: неправильно указан период
- Решение: поменяйте даты местами

#### Ошибки расчета

**"Ошибка при расчете: деление на ноль"**
- Причина: выработка или время работы = 0
- Решение: проверьте входные данные, убедитесь что блок работал

**"Ошибка при сохранении данных"**
- Причина: проблема с БД или дубликат записи
- Решение: проверьте, не вводились ли уже данные за эту дату

### Часто задаваемые вопросы (FAQ)

**Q: Как исправить ошибочно введенные данные?**

A: Выберите тип расчета "Перерасчет", введите корректные данные и сохраните.

---

**Q: Что делать, если счетчик обнулился (переполнение)?**

A: Система автоматически учитывает переполнение счетчика. Просто введите показания как обычно.

---

**Q: Можно ли удалить ошибочную запись?**

A: В текущей версии удаление записей недоступно. Используйте перерасчет для корректировки данных. Функция удаления будет добавлена в следующей версии.

---

**Q: Как часто нужно выполнять расчеты?**

A: 
- BAXTA, WYRABOTKA - ежедневно (в конце смены)
- PEREJEG - ежедневно
- O2GOL - еженедельно или по запросу

---

**Q: Что означает отрицательное значение ЭКСУТ?**

A: Отрицательное значение означает экономию топлива по данному показателю. Положительное - перерасход.

---

**Q: Как интерпретировать удельный расход (УР)?**

A: УР измеряется в г/кВт·ч. Типичные значения:
- 340-350 г/кВт·ч - отличный режим
- 350-360 г/кВт·ч - хороший режим
- 360-380 г/кВт·ч - удовлетворительный режим
- > 380 г/кВт·ч - требуется анализ и оптимизация

### Приложение Б: Формат даты NOME

Дата в формате NOME используется в таблице Perejeg:

```
NOME = ГГММДД

Примеры:
- 26 июня 2026 → 260626
- 1 января 2026 → 260101
- 31 декабря 2025 → 251231
```

Преобразование из NOME в DateTime и обратно:

```csharp
// NOME → DateTime
int nome = 260626;
int year = nome / 10000 + 2000;  // 26 → 2026
int month = (nome / 100) % 100;  // 06
int day = nome % 100;            // 26
DateTime date = new DateTime(year, month, day);

// DateTime → NOME
int nome = (date.Year % 100) * 10000 + date.Month * 100 + date.Day;
```

### Приложение Г: Глоссарий

**Блок (энергоблок)** - единица генерирующего оборудования, состоящая из котла и турбины.

**Вахта** - бригада операторов, работающая по графику сменности (обычно 4 вахты).

**Выработка электроэнергии** - количество произведенной электроэнергии за период.

**ДОП** - дополнительные потери, связанные с режимом работы.

**Дросселирование** - снижение давления пара перед турбиной, приводящее к потерям.

**КПД (коэффициент полезного действия)** - отношение полезной работы к затраченной энергии.

**Нагрузка** - мощность, выдаваемая блоком в данный момент.

**О2 (кислород)** - содержание кислорода в уходящих газах, показатель избытка воздуха.

**ПВД (подогреватель высокого давления)** - устройство для подогрева питательной воды паром.

**Перерасход топлива** - расход топлива выше нормативного.

**ПОП (давление острого пара)** - давление свежего пара перед турбиной.

**ПУГ** - потери с уходящими газами.

**Промперегрев** - повторный нагрев пара после частичного расширения в турбине.

**СН (собственные нужды)** - электроэнергия, потребляемая вспомогательным оборудованием станции.

**ТОП (температура острого пара)** - температура свежего пара.

**ТПВ (температура питательной воды)** - температура воды на входе в котел.

**ТПП (температура промперегрева)** - температура пара после промперегревателя.

**Удельный расход** - расход топлива на единицу выработанной электроэнергии (г/кВт·ч).

**Уходящие газы** - продукты сгорания топлива, выходящие из котла.

**ЭКСУТ** - экономия/перерасход условного топлива.
